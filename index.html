PAE Workbench

Building draft review evidence for AI-enabled decisions.

PAE Workbench helps evaluators define a use case, classify risk, map vendor and user evidence, generate tests and monitoring plans, and draft a residual-risk posture for accountable human review.

PAE is a practical framework and workbench for deciding what evidence is needed before relying on an AI-enabled use.

Evidence Map Vendor and user-side evidence, separated by source.
Gap List Open issues that must be resolved, conditioned, or accepted.
Tests and Monitoring Required checks, acceptance signals, metrics, and triggers.
Decision Brief A draft residual-risk package for accountable review.
Prototype boundary. PAE Workbench is an educational evaluation aid based on a working framework. It does not provide legal, regulatory, compliance, security, model-risk, procurement, or professional advice. Outputs are draft review artifacts, not approval, certification, validation, or proof of safety.
Suggested first tour: open the flagship demo, review Scope, Evidence, Tests, and Monitoring, then open Artifacts to see the draft brief and readiness review.

How to Use This Tool

Suggested workflow
  1. Start or load a case. Use the flagship Vendor LLM demo to see the framework end to end, use `Blank New Case` for a baseline evaluation, or use the Library when the case should start from a custom pattern.
  2. Define scope before judging risk. Record the business purpose, owner, users, system boundary, intended uses, and prohibited uses.
  3. Answer risk drivers. Use the risk-driver responses to set the review tier and surface missing information.
  4. Build the evidence record. Separate vendor evidence from user-side evidence, mark gaps, and avoid treating vendor material as proof of local fitness.
  5. Plan tests and monitoring. Convert evidence gaps and risk drivers into required tests, monitoring metrics, incident triggers, and reapproval conditions.
  6. Document the decision posture. Use the decision and artifacts tabs to produce a reviewable package, not an automatic pass/fail answer.

      How to Use This Tool

      Workbench guide

      Start Here

      Use the flagship demo as a three-minute tour.

      Open the Vendor LLM case, scan Scope, Evidence, Tests, and Monitoring, then open Artifacts to see the draft decision brief and readiness review.

      1. Scope

      Define the use before evaluating the tool.

      Begin with a blank new case unless a Library case pattern clearly fits. Record the business purpose, intended users, system boundary, data exposure, and prohibited uses.

      2. Risk and Evidence

      Use risk drivers to shape the evidence package.

      Answer the risk-driver questions, then collect evidence by source. Vendor documentation and local testing should remain separate so gaps are visible.

      3. Tests and Monitoring

      Turn gaps into review obligations.

      Add tests, acceptance signals, monitoring metrics, incident triggers, and reapproval conditions before relying on the generated decision artifacts.

      4. Review Package

      Export a reviewable record.

      Use decision and artifacts to draft the residual-risk posture, conditions, readiness review, decision brief, and artifact bundle for human review.
